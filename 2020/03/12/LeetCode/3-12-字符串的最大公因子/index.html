3.12-字符串的最大公因子

发表于 2020-03-12 | 分类于 LeetCode ， 每日一题

每日一题 字符串的最大公因子 双指针

题目

思路

这道题目的思路其实很简单。就是再寻找能累加形成两个字符串的最大公因子字符串。

所以，这里就有了两个条件：

  • 字符串的size一定是两个字符串size的公因子
  • 该公因子字符串最终能构成两个字符串

因此，我们有两种思路来求解：

暴力解法

  • 通过判定，先得到两个字符串之间的较小的那个。
  • 判定两个字符串的前面部分是否相等。
  • 搜寻可以被两边size整除的最大公因子。
  • 判定公因子字符串是否可以构成两个字符串。

数学解法

我们发现，两个字符串的关系，导致他们一定是可以前后叠加相等的。

根据题目我们知晓：

$S = m X \quad T = n X$

所以，我们可以得到的是：

$S+T = (m+n) X \quad T + S = (n + m)X$

因此，可以得到结论：

$S + T = T + S$。

这样，如果两个字符串不能前后叠加相等，那么就不可能有公因子字符串。

然后，求解 $m$ 和 $n$ 的最大公因子，就可以得到最大公因子字符串了。
